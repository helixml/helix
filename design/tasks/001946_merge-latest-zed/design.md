## Resolution Patterns (carry from 001864 and 001909)

### CI / GitHub workflows (`/.github/workflows/*.yml`)
Accept upstream (`git checkout --theirs`). Helix doesn't use Zed's CI.

### `Cargo.lock`
Accept upstream (`git checkout --theirs`). Will be regenerated by `cargo build --features external_websocket_sync` and the Helix-specific deps re-added automatically.

### `agent_panel.rs` / `conversation_view.rs` / `acp_thread.rs`
Always do a manual three-way merge. The Helix cfg-gated blocks reference upstream identifiers that may have been renamed. After resolving conflicts, **grep for old names** to catch references in unconflicted regions:

```bash
grep -rn "ActiveView\|set_active_view\|draft_threads\|background_threads" crates/agent_ui/src/
```

### Feature propagation chain
`zed/Cargo.toml` → `agent_ui/Cargo.toml` → `title_bar/Cargo.toml`. All three must propagate `external_websocket_sync`. Title bar's dep MUST be `optional = true` and the `[features]` section must include `external_websocket_sync = ["dep:external_websocket_sync"]` — otherwise the cfg-gated icon never renders.

### `rust-embed` workspace feature
Workspace `rust-embed` dep MUST keep both `include-exclude` AND `debug-embed`. Lost twice already (re-added in 001909). Without `debug-embed`, dev builds panic at startup.

### `wait_for_tools_ready`
Must use `cx.background_executor().timer(d).await` — NOT `smol::Timer::after`. `smol` was removed from the agent crate's deps by upstream PR #53603 (Apr 2026). If a regression slips in here, the build breaks.

## Notes & Carry-Overs from 001864 / 001909

- **`HeadlessConnection` is dead code** — referenced extensively in portingguide but only exists in `crates/agent_ui/src/acp/thread_view.rs`, which is not declared as a module. Active `from_existing_thread()` in `conversation_view.rs` uses `thread.read(cx).connection().clone()`. Don't waste time tracking trait method updates for it.
- **`ConnectedServerState` has 6 fields** (`connection`, `auth_state`, `active_id`, `threads`, `conversation`, `_connection_entry_subscription`) — no `history`. Portingguide entry on `history` is stale.
- **`AcpThreadEvent::Stopped` is a tuple variant** — `Stopped(StopReason)`. Pattern matches need `Stopped(_)`; emissions need an arg.
- **`title_bar` feature flag must be `optional = true`** — otherwise `#[cfg(feature = ...)]` evaluates false silently.
- **Auto-merged ≠ correct** — `git merge` may auto-resolve files where Helix cfg-gated code references identifiers that upstream renamed. Always grep after the merge.
- **`./script/clippy` over `cargo clippy`** — per `crates/zed/CLAUDE.md` build guideline.
- **E2E test has 12 phases now** (not 10 as some older parts of the porting guide may say). Phase 11 = Spectask routing, Phase 12 = Reconnect.
- **In-progress fork main pushes**: it is possible the user pushes new commits to fork main during the merge. If that happens, merge `origin/main` into the feature branch (small Cargo.lock conflict expected — take theirs) and rebuild.
