## Decisions taken

1. **Org-scoped only** â€” no project_id field. List view lives at
   `/orgs/{orgSlug}/sandboxes`.
2. **Public name "Sandbox"** â€” internal hydra-host registry methods renamed
   to `*SandboxInstance` to free the term. Existing `/admin/agent-sandboxes/*`
   routes kept as-is.
3. **Resource pinning** â€” every sandbox is 1 vCPU / 2GB RAM in v1. Stored on
   the row so we can vary per runtime later.
4. **TTL** â€” default 1h, set on create, extended via PATCH.
5. **No network policy** â€” default Docker bridge networking; user code can
   reach the public internet.
6. **No persistence** â€” DELETE tears the container down and forgets every
   cached command/log on the hydra side. No snapshots.

## Surface (REST)

```
GET    /api/v1/organizations/{org_id}/sandboxes              list
POST   /api/v1/organizations/{org_id}/sandboxes              create
GET    /api/v1/sandboxes/{id}                                get
PATCH  /api/v1/sandboxes/{id}                                update name/tags/ttl
DELETE /api/v1/sandboxes/{id}                                delete

POST   /api/v1/sandboxes/{id}/commands                       run command
GET    /api/v1/sandboxes/{id}/commands                       list commands
GET    /api/v1/sandboxes/{id}/commands/{cmd_id}              get command
GET    /api/v1/sandboxes/{id}/commands/{cmd_id}/logs         SSE log stream
POST   /api/v1/sandboxes/{id}/commands/{cmd_id}/kill         signal command

GET    /api/v1/sandboxes/{id}/files?path=...                 read raw bytes
PUT    /api/v1/sandboxes/{id}/files?path=...&mode=0644       write raw body
DELETE /api/v1/sandboxes/{id}/files?path=...&recursive=1     remove
GET    /api/v1/sandboxes/{id}/files/list?path=...            ls

GET    /api/v1/sandboxes/{id}/terminal                       websocket xterm
```

Auth: every route requires authenticated user; `loadAuthorizedSandbox` checks
membership in the sandbox's organization for non-list routes.

Frontend: `/orgs/{orgSlug}/sandboxes` (list) and
`/orgs/{orgSlug}/sandboxes/{id}` (detail with tabs Overview, Terminal,
Commands, Files).

## Runtimes shipped

- `ubuntu-desktop` â€” full desktop image, `HELIX_DISABLE_AGENT=1` skips the
  Zed/Qwen autoboot. Streamable display + xterm terminal in the UI.
- `headless-ubuntu` â€” plain `ubuntu:22.04` running `sleep infinity`. No GUI,
  no desktop, no agent â€” just a long-lived shell. Used by the SDK examples
  for reliable CI-style testing.

## Routes

All sandbox routes live under `/api/v1/organizations/{org_id}/...`. Cross-org
id-guessing is blocked by `loadAuthorizedSandbox` which verifies the URL's
`org_id` matches the sandbox row's `organization_id`.

## SDK

A Vercel-style JavaScript SDK ships at `examples/sdk/javascript`. Pure TS,
ESM-only, single dependency (`ws` for the terminal helper). Examples:
`create-and-run`, `file-roundtrip`, `list-and-cleanup`, `stream-logs` â€” all
verified end-to-end against the inner Helix instance with a real org +
API key.
