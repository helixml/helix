To check the status:
  kubectl get pods -l "app.kubernetes.io/name={{ include "helix-sandbox.name" . }}"

To view logs:
  kubectl logs -f -l "app.kubernetes.io/name={{ include "helix-sandbox.name" . }}"

To verify dockerd is running inside the sandbox:
  kubectl exec -it deploy/{{ include "helix-sandbox.fullname" . }} -- docker info

To check Hydra status:
  kubectl exec -it deploy/{{ include "helix-sandbox.fullname" . }} -- ls -la /var/run/hydra/

Troubleshooting:
  - If pods fail to start, check that privileged containers are allowed
  - Verify GPU nodes have the NVIDIA device plugin installed (for NVIDIA GPUs)
  - Check control plane URL and runner token are correct

Note: The sandbox requires privileged mode for Docker-in-Docker functionality.
Ensure your cluster's security policies allow privileged containers.
